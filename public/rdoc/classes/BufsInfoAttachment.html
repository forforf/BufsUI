Class BufsInfoAttachment
In: lib/bufs_info_attachment.rb
Parent: CouchRest::ExtendedDocument

Converts from BufsInfoDoc attachment format to closer to the metal couchrest/CouchDB attachment format. The reason this is needed is because CouchDB cannot support custom metadata for attachments. So custom metadata is held in the BufsInfoAttachment document. This document will also hold the attachments and its built in metadata (such as content-type and modified times 

 Attachment structure:
 attachments =>{ attachment_1 => { 'data1' => raw attachment data1,
                                   'md1' => combined attachment metadata1 },
                 attachment_2 => { 'data2' => raw attachment data2,
                                   'md2' => combined attachment metadata2 }
 }
